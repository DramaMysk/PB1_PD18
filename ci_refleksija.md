1. Kas notika, kad tests bija kļūdains?
Tad CI indikators paliek sarkans un tas nozime ka kaut kur ir kļūda un tas ir jālabo!
2. Kāpēc CI palīdz ātri pamanīt kļūdas?
Tapēc ka viņš katru reizi pārbauda kodu kad mēs taisam git push un var pamanīt kas tika izmainits un del kuras koda dalas ir kluda!
3. Kā DoD palīdz komandai?
Ir izdarīts plāns , pēc kura tu sekosi savam darbam un nedarisi liekas darbibas, tikai pēc plāna! Tev ir jaizdara noteiktas darbibas!
4. Kā mainījās tava attieksme pret git push ?
sapratu ka tas ir diezgan erti, kods parbaudas un saglabajas makoni, ar git push vispirms ir jasaglaba kods, pvienu dienu es rasktiju un domāju ka saglabāsu vēlāk, bet sanaca ka windows noķēra gļūku un izslēdzās un pus darba pazuda... tagad vairāk saglabāju un pielietošu git add / commit / push
